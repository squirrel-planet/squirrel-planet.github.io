New Bee Lang

警告：本编程语言会让你怀疑人生

New Bee Lang 是一门专为折磨程序员而设计的恶搞编程语言。它以蜜蜂为主题， 充满了对开发者的无情嘲讽和难以理解的语法规则。如果你想体验被蜜蜂支配的恐惧， 欢迎来到 New Bee Lang 的世界！

核心特性

New Bee Lang 不仅仅是一门编程语言，它更是一种生活态度

强制问候系统

程序必须以 hello New Bee Lang 开头，以 goodbye New Bee Lang 结尾。忘记问候？蜜蜂们会毫不留情地拒绝执行你的代码，并送上一段尖酸刻薄的批评。

plese 语句占比

你的代码中必须有 40% 到 50% 的代码行是 plese 语句。这是蜜蜂们用来维持世界平衡的方式，也是对你意志力的考验。每写五个语句，就必须插入一个 plese 语句。

业界领先的嘲讽系统

当你的代码出错时，蜜蜂们会打印出精心准备的嘲讽信息，从「这代码比我奶奶的食谱还难懂」到「你的键盘是不是被蜜蜂蜇过」，每一句都直击心灵。

语法规则

通往成功的必经之路，请仔细阅读

强制性问候与告别

每一个 New Bee Lang 程序都必须以 hello New Bee Lang 开头，以 goodbye New Bee Lang 结尾。这些语句必须一字不差，包括大小写。忘记问候？程序拒绝运行。忘记告别？程序拒绝运行。蜜蜂们对礼仪的要求就是这么严格。

plese 语句占比

plese 语句必须占所有有效代码行的 40% 到 50%。如果低于 40%，蜜蜂们会认为你不够礼貌。如果高于 50%，蜜蜂们会认为你只会说「请」而不会做实事。此外，不能连续出现超过 5 行非 plese 语句。

有效字符限制

New Bee Lang 只接受字母、数字和空格作为有效字符。这意味着你不能使用标点符号、括号、引号等任何其他字符。如果你胆敢使用这些「非法字符」，蜜蜂们会打印出一段专门针对该字符的嘲讽。唯一允许使用特殊字符的地方是在 set value 命令和 error 命令中。

注释支持

New Bee Lang 支持行内注释，使用 # 符号标记。# 之后的所有内容都将被视为注释，不会被解释执行。但请注意，# 符号本身不能出现在其他上下文中，它只用于注释。

命令参考

每个命令都与蜜蜂的生活习性相关，请仔细阅读

init honeycomb
初始化蜂巢，创建包含蜂后（queen bee）的蜂巢栈。每个程序的起点。
call
从蜂巢中取出一项，放入工作区。通常用于取出蜂巢顶部的蜜蜂。
lay egg
在工作区中的蜂后产下一枚卵。要求工作区顶部必须是蜂后。
sleep
让蜂巢顶部的蜜蜂成长。卵→幼蜂→普通蜜蜂
kill
移除蜂巢顶部的蜜蜂。但不能移除蜂后，否则程序立即终止。
all kill
清空整个蜂巢。不可逆的操作，请三思而后行。
all sleep
让蜂巢中所有的卵和幼蜂成长一次。批量操作节省时间。
go back
将工作区顶部的蜜蜂放回蜂巢。保持蜂巢的秩序。
set job worker bee
将蜂巢顶部的普通蜜蜂设置为工蜂。工蜂可以执行各种计算任务。
set value 'X'
为蜂巢顶部的蜜蜂设置一个单字符值。值必须用单引号包裹。
set value input
通过输入为蜂巢顶部的蜜蜂设置一个单字符值。
output
输出蜂巢顶部蜜蜂的值。如果该蜜蜂没有值，程序会报错。
plus
加法运算。蜂巢顶部两个数字相加，结果以新蜜蜂形式压入蜂巢。
reduce
减法运算。蜂巢顶部两个数字相减（第一个减第二个）。
ride
乘法运算。蜂巢顶部两个数字相乘。
except
整除运算。蜂巢顶部两个数字相除并取整。
loop start / loop end
循环结构。循环体内的代码会被重复执行。需配合 break 使用。
break
跳出当前循环。立即终止最内层的循环。
judgment start / judgment end
条件判断结构。检查蜂巢顶部蜜蜂的值是否为真。
give up
立即终止程序。打印告别信息并以错误码退出。
error
主动触发错误。允许「自定义」错误信息，但仍然终止程序。
plese
最独特的命令。表示「礼貌地请求」执行该命令。每5个命令必须插入一个。

错误处理

当你犯错时（这是必然会发生的），系统会打印出一段精心设计的嘲讽信息

🐝

嘲讽系统

错误信息通常包含随机选择的嘲讽语，如「这代码比我奶奶的食谱还难懂！」或「如果代码能哭，它现在正在嚎啕大哭。」这些话语旨在让你感到自己的渺小和无能。

然后是具体的错误描述，指出你在哪里犯了什么错误。最后是一段「鼓励」话语，如「继续加油！你快要理解了... 才怪！」
